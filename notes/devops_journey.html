Глава 1. Установка и настройка Fedora 44

Fedora 44 была установлена как основная рабочая система на планшете-ноутбуке Ninkear T40 с процессором Intel N100 (Alder Lake ULX), 16 ГБ RAM, SSD 512 ГБ.

1.1 Создание установочной флешки

Поскольку установка производилась с MacBook (macOS 12.7.6), использовалась команда dd через терминал. Fedora Media Writer не поддерживал данную версию macOS.

# Найти флешку в системе

diskutil list

# Флешка определилась как /dev/disk2 (124 ГБ, external)

# ISO образ смонтировался как /dev/disk3

# Размонтировать флешку

diskutil unmountDisk /dev/disk2

# Записать образ на флешку

sudo dd if=/dev/disk3 of=/dev/rdisk2 bs=4m

# rdisk2 — raw режим, работает значительно быстрее чем disk2

⚠️ Использование rdisk вместо disk ускоряет запись в несколько раз на macOS. Не перепутайте номер диска — запись на системный диск уничтожит macOS.

Глава 2. Настройка SSH

SSH настраивался на двух машинах: Fedora (рабочая станция) и РЕД ОС (виртуальный сервер). Также настраивалось подключение с телефона Android через Termius.

2.1 Настройка SSH сервера на Fedora

sudo dnf install -y openssh-server

sudo systemctl enable --now sshd

# Настройка конфига

sudo nano /etc/ssh/sshd_config

# Port 2222 — нестандартный порт для безопасности

# PermitRootLogin no — запрет входа под root

# MaxAuthTries 3 — максимум попыток ввода пароля

# ClientAliveInterval 600 — таймаут неактивной сессии

# Разрешить порт в SELinux (обязательно для Fedora/RHEL!)

sudo semanage port -a -t ssh_port_t -p tcp 2222

# Открыть порт в firewall

sudo firewall-cmd --zone=FedoraWorkstation --permanent --add-port=2222/tcp

sudo firewall-cmd --reload

❌ Ошибка: SSH не запускался после смены порта — status=255/EXCEPTION

✅ SELinux блокировал нестандартный порт. Решение: semanage port -a -t ssh_port_t -p tcp 2222

2.2 Настройка SSH ключей

# Генерация ключа на клиентской машине

# -t ed25519 — современный алгоритм (быстрее и безопаснее RSA)

# -C — комментарий для идентификации ключа

ssh-keygen -t ed25519 -C "redos-server"

# Копирование публичного ключа на сервер

# -i — указать файл ключа

# -p — нестандартный порт

ssh-copy-id -i ~/.ssh/id_ed25519.pub -p 2222 adminos@192.168.56.101

# Создать конфиг для удобного подключения

nano ~/.ssh/config

Host redos

HostName 192.168.56.101

User adminos

Port 2222

IdentityFile ~/.ssh/id_ed25519

# Теперь подключение одной командой

ssh redos

❌ Ошибка: ssh-keygen -t ed25519 -c (маленькая c) — ошибка 'too many arguments'

✅ -C (заглавная C) — комментарий. -c (маленькая c) — другая функция, требует доп. аргументов

2.3 Добавление SSH ключа на GitHub

# Просмотр публичного ключа для копирования

cat ~/.ssh/id_ed25519.pub

# Проверить подключение к GitHub

ssh -T git@github.com

# Ожидаемый ответ: Hi khimeran! You've successfully authenticated

💡 При первом подключении к любому SSH серверу появляется предупреждение о неизвестном хосте. Нужно ввести 'yes' — fingerprint сохранится в ~/.ssh/known_hosts и больше не будет спрашивать.

2.4 Настройка fail2ban

sudo dnf install -y fail2ban

# Создать конфиг /etc/fail2ban/jail.local

[DEFAULT]

bantime = 3600 # блокировка на 1 час

findtime = 600 # период наблюдения 10 минут

maxretry = 3 # максимум попыток

ignoreip = 127.0.0.1/8 192.168.56.0/24 # не блокировать локальную сеть

[sshd]

enabled = true

port = 2222

logpath = /var/log/secure

sudo systemctl enable --now fail2ban

sudo fail2ban-client status sshd

РЕД ОС основана на RHEL — SELinux включён по умолчанию в режиме Enforcing. Это источник многих неожиданных ошибок доступа.

Команда Описание

sudo setenforce 0 Временно перевести в Permissive (только логи)

4.4 Layer Cache — оптимизация сборки

Каждая команда в Dockerfile создаёт слой. Слои кэшируются. Правило: сначала то что меняется редко, потом то что меняется часто.

# ПЛОХО — код меняется, зависимости пересобираются каждый раз

COPY . /app

RUN pip install -r requirements.txt

# ХОРОШО — зависимости кэшируются отдельно

COPY requirements.txt . # меняется редко → кэш

RUN pip install -r requirements.txt # берётся из кэша

COPY . /app # меняется часто → пересобирается

💡 Правильный порядок слоёв сокращает время сборки с 5 минут до 10 секунд при изменении только кода.

4.6 SELinux и Docker на Fedora

Docker монтирует файлы с хоста в контейнеры через volumes. SELinux блокирует этот доступ если у файлов неправильный контекст безопасности.

# Исправить SELinux контекст для папки монтируемой в Docker

sudo chcon -Rt svirt_sandbox_file_t ~/etr_study_api/ops/

❌ Ошибка: permission denied при монтировании конфигов Prometheus и Nginx

✅ sudo chcon -Rt svirt_sandbox_file_t папка/ — изменить SELinux контекст

6.6 Идемпотентность — главный принцип Ansible

Идемпотентность — многократный запуск playbook даёт тот же результат что и однократный. Ansible проверяет текущее состояние и вносит изменения только если нужно.

# НЕ идемпотентно — каждый запуск добавляет строку

- shell: echo '127.0.0.1 host' >> /etc/hosts

# Идемпотентно — проверяет наличие строки

- lineinfile:

path: /etc/hosts

line: '127.0.0.1 host'

state: present

7.5 Ошибки из практики

❌ Ошибка: ветка создалась как 'master' вместо 'main'

✅ git branch -m master main — переименовать. git config --global init.defaultBranch main — исправить для будущих репо

❌ Ошибка: создали файл READMI.md вместо README.md

✅ git mv READMI.md README.md — переименовать через Git (сохраняет историю)

❌ Ошибка: попытка git commit без предварительного git add — 'индекс пуст'

✅ Всегда: изменить файл → git add → git commit

❌ Ошибка: *.env в .gitignore не защищает файл .env

✅ *.env игнорирует файлы с расширением .env (prod.env). Для файла .env нужно писать именно .env

❌ Ошибка: gitignore без точки — Git не видит файл

✅ mv gitignore .gitignore — точка в начале делает файл скрытым в Linux

❌ Ошибка: git merge branchA (без дефиса) — 'не является тем что можно слить'

✅ Дефис важен: branch-A и branchA — разные ветки

8.2 firewalld — управление firewall

# Показать все правила активной зоны

sudo firewall-cmd --list-all

# Открыть порт навсегда

# --permanent — сохранить после перезагрузки

# --reload — применить изменения

sudo firewall-cmd --zone=FedoraWorkstation --permanent --add-port=8000/tcp

sudo firewall-cmd --reload

# Открыть сервис по имени

sudo firewall-cmd --permanent --add-service=http

# Закрыть порт

sudo firewall-cmd --permanent --remove-port=8000/tcp

⚠️ --permanent без --reload не применяется! Всегда выполняйте firewall-cmd --reload после --permanent

8.3 Firewall на разных дистрибутивах

Дистрибутив Firewall Открыть порт
Fedora / RHEL / РЕД ОС firewalld firewall-cmd --add-port=80/tcp
Ubuntu (новые) ufw ufw allow 80/tcp
Ubuntu (старые) iptables iptables -A INPUT -p tcp --dport 80 -j ACCEPT
Debian nftables nft add rule inet filter input tcp dport 80 accept
