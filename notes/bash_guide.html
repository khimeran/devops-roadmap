1. Основы — с чего начинается любой скрипт

1.1 Shebang — первая строка

Первая строка скрипта указывает какой интерпретатор использовать. Без неё скрипт выполняется текущей оболочкой что может привести к неожиданному поведению.

#!/bin/bash # стандартный Bash

#!/usr/bin/env bash # лучший вариант — ищет bash в PATH

#!/bin/sh # POSIX sh — меньше возможностей но работает везде

💡 #!/usr/bin/env bash предпочтительнее потому что не зависит от расположения bash в системе. На некоторых системах bash находится в /usr/local/bin/bash а не в /bin/bash.

1.2 Три защитных флага — всегда ставьте в начало

Без этих флагов скрипт продолжает работу даже при ошибках что может привести к катастрофическим последствиям.

#!/usr/bin/env bash

set -euo pipefail

# set -e — выйти при любой ошибке (exit on error)

# set -u — ошибка при использовании необъявленной переменной

# set -o pipefail — ошибка если упала любая команда в pipeline

Почему каждый флаг важен:

# Без set -e:

rm -rf /важная/папка # ошибка — папка не существует

echo 'продолжаем...' # выполнится! Потенциально опасно

# С set -e:

rm -rf /важная/папка # ошибка — скрипт ОСТАНАВЛИВАЕТСЯ

echo 'продолжаем...' # НЕ выполнится

# Без set -u:

echo $UNDEFINED_VAR # выведет пустую строку — нет ошибки!

# С set -u:

echo $UNDEFINED_VAR # ошибка: unbound variable

# Без set -o pipefail:

cat несуществующий | grep text # cat упал, но grep вернул 0

echo $? # 0 — скрипт думает что всё хорошо!

# С set -o pipefail:

cat несуществующий | grep text # ошибка всего pipeline

❌ Ошибка из практики: THRESHHOLD=80 объявили переменную, а обращались к $THRESHOLD (другое имя). Без set -u — молчаливая пустая строка. С set -u — сразу ошибка.

✅ Всегда добавляйте set -euo pipefail — это спасёт от трудноуловимых багов

1.3 Права на выполнение

# Сделать файл исполняемым

chmod +x script.sh # добавить бит execute всем

chmod 755 script.sh # rwxr-xr-x — стандарт для скриптов

chmod 700 script.sh # rwx------ — только владелец

# Запуск скрипта

./script.sh # из текущей папки

bash script.sh # явно через bash (без chmod +x)

/home/kirill/script.sh # абсолютный путь

2. Переменные — полное руководство

2.1 Объявление и использование

# Объявление — БЕЗ пробелов вокруг =

имя='Kirill' # строка в одинарных кавычках

фраза="Привет, $имя" # строка в двойных кавычках (подстановка)

число=42 # число (в Bash всё строки!)

пустая='' # пустая строка

# Использование — $ перед именем

echo $имя # Kirill

echo "$имя" # Kirill (в кавычках — безопаснее)

echo ${имя} # Kirill (фигурные скобки — явные границы)

❌ Ошибка: имя = 'Kirill' — пробелы вокруг = вызывают ошибку 'command not found'

✅ имя='Kirill' — никаких пробелов вокруг знака равенства

2.2 Одинарные vs двойные кавычки

Кавычки Подстановка переменных Спецсимволы Когда использовать
'одинарные' НЕТ — буквально НЕТ — буквально Когда нужен точный текст
"двойные" ДА — $VAR заменяется ДА — \n, \t работают Когда нужны переменные
без кавычек ДА ДА Избегайте — сплит по пробелам

name='World'

echo 'Hello $name' # Hello $name — буквально

echo "Hello $name" # Hello World — подстановка

2.4 Подстановка команд

Выполняет команду и подставляет её вывод как значение переменной.

# Современный синтаксис — рекомендуется

дата=$(date +"%Y-%m-%d")

пользователь=$(whoami)

кол_файлов=$(ls -1 | wc -l)

# Старый синтаксис — избегайте

дата=`date +"%Y-%m-%d"` # backticks — нельзя вкладывать

# Вложенная подстановка — только с $()

размер=$(du -sh $(pwd) | cut -f1)

2.5 Арифметика

a=10

b=3

# Способ 1 — $(( )) — рекомендуется

сумма=$((a + b)) # 13

разность=$((a - b)) # 7

произведение=$((a * b)) # 30

частное=$((a / b)) # 3 (целочисленное!)

остаток=$((a % b)) # 1

степень=$((a ** 2)) # 100

# Инкремент и декремент

((a++)) # a = a + 1

((a--)) # a = a - 1

((a += 5)) # a = a + 5

# Способ 2 — expr (устаревший)

сумма=$(expr $a + $b)

# Дробные числа — только через bc

результат=$(echo "scale=2; 10 / 3" | bc) # 3.33

2.6 Строковые операции

str='Hello World'

# Длина строки

echo ${#str} # 11

# Срез строки ${var:начало:длина}

echo ${str:0:5} # Hello

echo ${str:6} # World (до конца)

echo ${str: -5} # World (с конца)

# Замена ${var/старый/новый}

echo ${str/World/Bash} # Hello Bash (первое вхождение)

echo ${str//l/L} # HeLLo WorLd (все вхождения)

# Удаление префикса/суффикса

файл='backup_2026.tar.gz'

echo ${файл%.tar.gz} # backup_2026 (удалить суффикс)

echo ${файл#backup_} # 2026.tar.gz (удалить префикс)

# Верхний/нижний регистр (Bash 4+)

echo ${str^^} # HELLO WORLD

echo ${str,,} # hello world

2.7 Значения по умолчанию

# ${var:-default} — использовать default если var пустая или не задана

имя=${1:-'Гость'}

echo "Привет, $имя" # если $1 не передан — 'Гость'

# ${var:=default} — присвоить default если var пустая

порт=${PORT:=8080}

# ${var:?сообщение} — ошибка если var пустая

db_url=${DATABASE_URL:?'DATABASE_URL не задана!'}

# ${var:+значение} — использовать значение если var НЕ пустая

debug=${DEBUG:+' --verbose'}

2.8 Массивы

# Объявление массива

фрукты=('яблоко' 'банан' 'вишня')

# Доступ к элементам

echo ${фрукты[0]} # яблоко (индекс с 0)

echo ${фрукты[1]} # банан

echo ${фрукты[-1]} # вишня (последний)

# Все элементы

echo ${фрукты[@]} # яблоко банан вишня

# Длина массива

echo ${#фрукты[@]} # 3

# Добавить элемент

фрукты+=('груша')

# Перебор массива

for фрукт in "${фрукты[@]}"; do

echo "Фрукт: $фрукт"

done

3. Условия и сравнения

3.1 Синтаксис if

if [[ условие ]]; then

# выполнить если истинно

elif [[ другое_условие ]]; then

# выполнить если второе истинно

else

# выполнить если ничего не совпало

fi

⚠️ Пробелы внутри [[ ]] обязательны! [[$VAR -eq 5]] — ошибка. [[ $VAR -eq 5 ]] — правильно.

3.2 [ ] vs [[ ]] vs (( ))

Тип Назначение Особенности
[ ] POSIX тест — старый Ломается от пробелов в переменных. Нужны кавычки
[[ ]] Bash тест — современный Безопасен. Поддерживает && || =~ для regex
(( )) Арифметическое условие Для сравнения чисел. Не нужен -eq -lt

# [ ] — старый способ, нужны кавычки

[ "$var" = 'hello' ]

# [[ ]] — современный, безопаснее

[[ $var == 'hello' ]]

[[ $var =~ ^[0-9]+$ ]] # regex проверка

# (( )) — для чисел

(( a > 5 ))

(( a >= b && c < 10 ))

3.3 Операторы сравнения

Числа Строки Файлы Значение
-eq == -f равно / равно / это файл
-ne != -d не равно / не равно / это директория
-gt > -e больше / больше / существует
-lt < -r меньше / меньше / доступен для чтения
-ge >= -w больше или равно / / доступен для записи
-le <= -x меньше или равно / / исполняемый
-z -s / строка пустая / файл не пустой
-n / строка не пустая /

3.4 Логические операторы

# И (AND)

[[ $a -gt 0 && $b -gt 0 ]]

[[ $a -gt 0 ]] && [[ $b -gt 0 ]]

# ИЛИ (OR)

[[ $a -gt 0 || $b -gt 0 ]]

# НЕ (NOT)

[[ ! -f /etc/passwd ]]

! command # если команда завершилась с ошибкой

3.5 Case — многозначное ветвление

case "$переменная" in

'значение1')

echo 'Первый случай'

;; # обязательный разделитель

'значение2'|'значение3') # несколько вариантов через |

echo 'Второй или третий'

;;

start|START) # регистр важен

echo 'Запуск'

;;

*) # * — любое другое значение

echo 'Неизвестное значение'

;;

esac

4. Циклы

4.1 For — перебор элементов

# По списку значений

for item in one two three; do

echo "Элемент: $item"

done

# По диапазону чисел

for i in {1..10}; do

echo "Число: $i"

done

# Диапазон с шагом

for i in {0..100..10}; do # 0 10 20 30...100

echo $i

done

# C-стиль (как в C/Java)

for ((i=0; i<10; i++)); do

echo $i

done

# По файлам

for file in /var/log/*.log; do

echo "Лог файл: $file"

wc -l "$file"

done

# По выводу команды

for user in $(cut -d: -f1 /etc/passwd); do

echo "Пользователь: $user"

done

# По аргументам скрипта

for arg in "$@"; do

echo "Аргумент: $arg"

done

4.2 While — пока условие истинно

# Базовый while

count=0

while [[ $count -lt 5 ]]; do

echo "Итерация: $count"

((count++))

done

# Чтение файла построчно

while IFS= read -r line; do

echo "Строка: $line"

done < /etc/hosts

# IFS= — не разбивать по пробелам

# -r — не интерпретировать backslash

# Чтение вывода команды построчно

while IFS= read -r line; do

echo "$line"

done < <(df -h)

# < <() — process substitution

# Бесконечный цикл

while true; do

echo 'Жду...'

sleep 5

done

4.3 Until — пока условие ложно

# Противоположность while

count=0

until [[ $count -ge 5 ]]; do

echo $count

((count++))

done

4.4 Управление циклами

break # выйти из цикла

continue # перейти к следующей итерации

# Пример с break

for i in {1..100}; do

if [[ $i -eq 10 ]]; then

break # остановить на 10

fi

echo $i

done

# Пример с continue

for i in {1..10}; do

if [[ $((i % 2)) -eq 0 ]]; then

continue # пропустить чётные

fi

echo $i # только нечётные

done

5. Функции — подробное руководство

5.1 Объявление функций

В Bash функции объявляются двумя способами — оба равнозначны:

# Способ 1 — с ключевым словом function

function имя_функции() {

# тело функции

}

# Способ 2 — без ключевого слова (рекомендуется)

имя_функции() {

# тело функции

}

⚠️ Функцию нужно объявить ДО её вызова. В отличие от некоторых языков Bash не поднимает объявления функций.

5.2 Аргументы функций

В сигнатуре функции НЕТ параметров — только пустые скобки. Аргументы доступны через позиционные переменные $1, $2, $3...

greet() {

local name=$1 # первый аргумент

local age=$2 # второй аргумент

echo "Привет, $name! Тебе $age лет."

}

# Вызов — аргументы через пробел

greet 'Kirill' 25

# Привет, Kirill! Тебе 25 лет.

Переменная Значение внутри функции
$0 Имя скрипта (НЕ функции)
$1 Первый аргумент функции
$2 Второй аргумент функции
$@ Все аргументы как отдельные строки
$* Все аргументы как одна строка
$# Количество аргументов функции

5.3 Локальные переменные — local

Без local переменная становится глобальной и может перезаписать переменную с тем же именем в основном скрипте.

name='Глобальный'

# БЕЗ local — опасно

bad_function() {

name='Локальный' # перезаписывает глобальную!

}

# С local — безопасно

good_function() {

local name='Локальный' # видна только здесь

}

bad_function

echo $name # Локальный — глобальная перезаписана!

good_function

echo $name # Глобальный — не тронута

💡 Всегда используйте local для переменных внутри функций. Это предотвращает трудноуловимые баги когда две функции случайно используют одно имя переменной.

5.4 Возврат значений

В Bash функция может вернуть только числовой код (0-255). Для возврата строк используется echo + подстановка команды $().

# return — только числовой код

# 0 = успех, 1-255 = ошибка

check_file() {

local file=$1

if [[ -f "$file" ]]; then

return 0 # успех

else

return 1 # ошибка

fi

}

# Использование кода возврата

if check_file '/etc/passwd'; then

echo 'Файл существует'

else

echo 'Файл не найден'

fi

# Возврат СТРОКИ — через echo + $()

get_timestamp() {

echo $(date +"%Y-%m-%d %H:%M:%S")

}

# Захват возвращаемой строки

timestamp=$(get_timestamp)

echo "Время: $timestamp"

5.5 Функции vs Python — сравнение

Аспект Python Bash
Сигнатура def func(a, b, c): func() { }
Параметры Явно в скобках Всегда пустые скобки
Доступ к аргументам По имени: a, b По позиции: $1, $2
Именованный вызов func(name='Kirill') ❌ Невозможно
Локальные переменные По умолчанию Нужно писать local
Возврат значения return любой тип return только 0-255
Возврат строки return 'текст' echo 'текст' + $()
Вложенные функции Поддерживаются Поддерживаются

5.6 Продвинутые техники функций

# Функция с проверкой аргументов

create_backup() {

# Проверить что аргументы переданы

if [[ $# -lt 2 ]]; then

echo "Использование: create_backup <source> <dest>" >&2

return 1

fi

local source=$1

local dest=$2

local timestamp=$(date +"%Y%m%d_%H%M%S")

tar -czf "${dest}/backup_${timestamp}.tar.gz" "$source"

}

# Рекурсивная функция

factorial() {

local n=$1

if [[ $n -le 1 ]]; then

echo 1

return

fi

local prev=$(factorial $((n - 1)))

echo $((n * prev))

}

echo $(factorial 5) # 120

6. Ввод, вывод и перенаправление

6.1 Стандартные потоки

Поток Номер Описание Перенаправление
stdin 0 Стандартный ввод (клавиатура) < файл
stdout 1 Стандартный вывод (результат) > файл или >> файл
stderr 2 Поток ошибок 2> файл

6.2 Перенаправление

# Запись в файл (перезаписать)

echo 'текст' > файл.txt

# Добавить в конец файла

echo 'текст' >> файл.txt

# Перенаправить ошибки в файл

команда 2> errors.log

# Перенаправить и вывод и ошибки в один файл

команда > output.log 2>&1

команда &> output.log # краткая форма

# Выбросить вывод (не показывать)

команда > /dev/null

команда > /dev/null 2>&1 # выбросить всё

# Читать из файла

while IFS= read -r line; do

echo "$line"

done < файл.txt

6.3 Pipe — конвейер команд

# Передать вывод одной команды на вход другой

ps aux | grep nginx

cat /etc/passwd | cut -d: -f1 | sort

df -h | tail -n +2 | awk '{print $5}'

# tee — вывести И записать в файл

команда | tee файл.log # перезаписать

команда | tee -a файл.log # добавить

команда | tee файл.log | grep error # дальше в pipe

# Многострочный ввод

cat << 'EOF'

name='Kirill'

7. Обработка текста — grep, sed, awk

7.1 grep — поиск по шаблону

# Базовый поиск

grep 'nginx' /var/log/syslog

# Полезные флаги

grep -i 'error' файл # -i игнорировать регистр

grep -r 'TODO' ./src/ # -r рекурсивно по папке

grep -n 'error' файл # -n показать номера строк

grep -v 'DEBUG' файл # -v инвертировать (исключить)

grep -c 'error' файл # -c подсчитать количество

grep -l 'error' *.log # -l показать только имена файлов

grep -A 3 'error' файл # -A 3 показать 3 строки ПОСЛЕ

grep -B 3 'error' файл # -B 3 показать 3 строки ДО

grep -E 'error|warning' # -E расширенный regex

# Regex примеры

grep '^ERROR' файл # строки начинающиеся с ERROR

grep 'ERROR$' файл # строки заканчивающиеся на ERROR

grep '[0-9]\{3\}' файл # три цифры подряд

grep -E '^[0-9]+\.' файл # строки начинающиеся с числа и точки

7.2 sed — потоковый редактор

# Замена текста

sed 's/старый/новый/' файл # первое вхождение в строке

sed 's/старый/новый/g' файл # все вхождения (g=global)

sed 's/старый/новый/gi' файл # игнорировать регистр

# Редактировать файл на месте

sed -i 's/старый/новый/g' файл # изменить файл

sed -i.bak 's/старый/новый/g' файл # с резервной копией .bak

# Удаление строк

sed '/pattern/d' файл # удалить строки с pattern

sed '5d' файл # удалить 5-ю строку

sed '1,3d' файл # удалить строки 1-3

# Вывод определённых строк

sed -n '5p' файл # показать 5-ю строку

sed -n '1,10p' файл # показать строки 1-10

# Вставка строк

sed '3a\новая строка' файл # добавить после 3-й

sed '3i\новая строка' файл # вставить перед 3-й

7.3 awk — обработка таблиц

# Базовый синтаксис: awk '{действие}' файл

# Поля разделены пробелом по умолчанию

# $1 = первое поле, $2 = второе, $NF = последнее

awk '{print $1}' файл # первое поле каждой строки

awk '{print $1, $3}' файл # первое и третье

awk '{print NR, $0}' файл # номер строки + вся строка

# Другой разделитель

awk -F: '{print $1}' /etc/passwd # разделитель :

awk -F, '{print $2}' data.csv # разделитель ,

# Условия

awk '$3 > 100 {print $1}' файл # если 3-е поле > 100

awk '/error/ {print}' файл # строки содержащие error

awk 'NR > 1 {print}' файл # пропустить первую строку

# Вычисления

awk '{sum += $3} END {print sum}' файл # сумма 3-го поля

awk 'END {print NR}' файл # количество строк

# Реальные примеры

df -h | awk '{print $5}' | tr -d '%' # заполненность дисков

ps aux | awk '{print $1, $2, $11}' # user PID command

cat /etc/passwd | awk -F: '{print $1}' # список пользователей

9. Шаблоны и готовые скрипты

9.1 Шаблон — основа любого скрипта

#!/usr/bin/env bash

set -euo pipefail

# ============================================

# Описание скрипта

# Использование: ./script.sh [аргументы]

# ============================================

# Константы

readonly SCRIPT_DIR=$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)

readonly SCRIPT_NAME=$(basename "$0")

readonly LOG_FILE="/tmp/${SCRIPT_NAME%.sh}.log"

# Функция логирования

log() {

local level=$1; shift

local message="$*"

local timestamp=$(date +"%Y-%m-%d %H:%M:%S")

echo "[$timestamp] [$level] $message" | tee -a "$LOG_FILE"

}

# Функция завершения с ошибкой

die() {

log 'ERROR' "$*"

exit 1

}

# Функция очистки

cleanup() {

log 'INFO' 'Завершение работы'

}

trap cleanup EXIT

# Проверка зависимостей

check_deps() {

local deps=('docker' 'curl' 'jq')

for dep in "${deps[@]}"; do

if ! command -v "$dep" &>/dev/null; then

die "Не найдена зависимость: $dep"

fi

done

}

# Главная функция

main() {

log 'INFO' "Запуск $SCRIPT_NAME"

check_deps

# Ваш код здесь

log 'INFO' "$SCRIPT_NAME завершён успешно"

}

main "$@"

9.2 Скрипт мониторинга системы

#!/usr/bin/env bash

set -euo pipefail

THRESHOLD_CPU=80

THRESHOLD_MEM=80

THRESHOLD_DISK=85

check_cpu() {

local cpu=$(top -bn1 | grep 'Cpu(s)' | awk '{print $2}' | cut -d. -f1)

if [[ $cpu -ge $THRESHOLD_CPU ]]; then

echo "WARN: CPU = ${cpu}%"

else

echo "OK: CPU = ${cpu}%"

fi

}

check_memory() {

local used=$(free | grep Mem | awk '{printf "%.0f", $3/$2*100}')

if [[ $used -ge $THRESHOLD_MEM ]]; then

echo "WARN: RAM = ${used}%"

else

echo "OK: RAM = ${used}%"

fi

}

check_disk() {

while IFS= read -r line; do

local usage=$(echo "$line" | awk '{print $5}' | tr -d '%')

local mount=$(echo "$line" | awk '{print $6}')

local dev=$(echo "$line" | awk '{print $1}')

[[ "$dev" == tmpfs* ]] && continue

if [[ $usage -ge $THRESHOLD_DISK ]]; then

echo "WARN: Диск $mount = ${usage}%"

else

echo "OK: Диск $mount = ${usage}%"

fi

done < <(df -h | tail -n +2)

}

echo '=== Мониторинг системы ==='

echo "Время: $(date '+%Y-%m-%d %H:%M:%S')"

echo '---'

check_cpu

check_memory

check_disk

#!/usr/bin/env bash

FAILED=$((FAILED + 1))

check_service "$service"

exit 1

else

echo 'Все сервисы работают'

fi

9.5 Скрипт деплоя приложения

#!/usr/bin/env bash

set -euo pipefail

APP_DIR=${APP_DIR:-"$HOME/app"}

COMPOSE_FILE="$APP_DIR/docker-compose.yml"

LOG_FILE='/tmp/deploy.log'

log() { echo "[$(date '+%H:%M:%S')] $*" | tee -a "$LOG_FILE"; }

die() { log "ERROR: $*"; exit 1; }

start() {

log 'Запуск стека...'

docker compose -f "$COMPOSE_FILE" up -d

health_check

}

stop() {

log 'Остановка стека...'

docker compose -f "$COMPOSE_FILE" down

}

health_check() {

local url=${HEALTH_URL:-'http://localhost:80/live'}

local attempts=5

local n=0

while [[ $n -lt $attempts ]]; do

curl -sf "$url" &>/dev/null && { log '✅ Сервис доступен'; return 0; }

((n++))

log "Попытка $n/$attempts..."

sleep 3

done

die 'Сервис не отвечает!'

}

case "${1:-help}" in

start) start ;;

stop) stop ;;

restart) stop; sleep 2; start ;;

status) docker compose -f "$COMPOSE_FILE" ps ;;

logs) docker compose -f "$COMPOSE_FILE" logs --tail 50 ;;

health) health_check ;;

*)

echo "Использование: $0 {start|stop|restart|status|logs|health}"

exit 0 ;;

esac

#!/usr/bin/env bash

set -euo pipefail

LOG_FILE=${1:-'/var/log/nginx/access.log'}

TOP_N=${2:-10}

[[ -f "$LOG_FILE" ]] || { echo "Файл не найден: $LOG_FILE"; exit 1; }

10. Типичные ошибки и их решение

❌ Ошибка: command not found при объявлении переменной

Причина: пробелы вокруг знака =

имя = 'Kirill' # ошибка — пробелы

✅ имя='Kirill' # правильно — без пробелов

❌ Ошибка: [: ожидается унарный оператор

Причина: переменная пустая или содержит пробелы, используются [ ] вместо [[ ]]

if [ $UNDEFINED -eq 5 ]; then # ошибка

✅ if [[ $UNDEFINED -eq 5 ]]; then # правильно

❌ Ошибка: кириллица в именах переменных не работает с set -u

Причина: Bash не поддерживает кириллицу в именах переменных

ПОРОГ=80 # может работать без set -u, ломается с ним

✅ THRESHOLD=80 # всегда используйте латиницу в именах переменных

❌ Ошибка: скрипт продолжает работу после ошибки

Причина: не установлены защитные флаги

✅ Добавьте в начало: set -euo pipefail

❌ Ошибка: переменная в функции перезаписывает глобальную

Причина: не использован local

func() { count=0; } # перезаписывает глобальный count

✅ func() { local count=0; } # изолировано внутри функции

❌ Ошибка: функция не найдена

Причина: функция вызвана до её объявления

main # вызов

main() { echo 'hello'; } # объявление ПОСЛЕ — ошибка

✅ Объявляйте функции ДО их вызова. Рекомендуется: все функции сверху, вызов main в конце

❌ Ошибка: $() не работает внутри одинарных кавычек

echo '$(date)' # выведет буквально $(date)

✅ echo "$(date)" # двойные кавычки — подстановка работает
